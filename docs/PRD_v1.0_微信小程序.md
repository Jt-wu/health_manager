# PRD v1.0（微信小程序）

## 1. 产品目标与范围

### 目标

- 让用户用一张照片完成“一餐多菜”的饮食记录。
- 自动识别并先自动估算营养与份量，用户可选微调（大中小/手掌参照）。
- 基于目标（减脂/高血压/高尿酸/脂肪肝/高血脂/抗炎）输出温和陪伴式建议。
- 每周输出按目标定制的周报（主目标 3 指标）。
- Plus 解锁：详细分析 + 周报深度洞察 + 更省事（更少追问）。

### MVP 不做（但预留）

- 饮水/运动/睡眠。
- 品牌/商家菜品库。
- 条码。
- 设备数据自动同步。

---

## 2. 用户故事（MVP 必须支持）

1. 作为用户，我想快速建档并选择改善目标，不想被复杂表单卡住。
2. 作为用户，我拍一张包含多菜的照片，系统自动识别并给出建议，我一键保存。
3. 作为用户，我觉得份量不准时，能用大中小或手掌参照微调，系统立即更新营养。
4. 作为高血压用户，我能看到本周钠相关表现与可执行建议。
5. 作为付费用户，我能看到“为什么 + 成分 + 替代方案 + 洞察”，并更省事。

---

## 3. 信息架构（页面与导航）

底部 Tab 建议：首页 / 记录 / 周报 / 我的

- **首页**：今日概览 + 一键拍照。
- **记录**：按天查看、按餐次列表、复用历史。
- **周报**：本周报告（主目标 3 指标）+ Plus 洞察入口。
- **我的**：档案/目标、指标录入、Plus 订阅、设置与协议。

---

## 4. 页面规格（字段、状态、文案、验收）

### 4.1 登录 & 合规确认

**目的**：微信登录、确认协议。

**控件**：微信授权登录按钮。

**勾选**：用户协议、隐私政策、免责声明（必勾）。

**验收**：

- 未勾选不能继续。
- 成功拿到 openid/unionid 并创建 User。

**默认免责声明文案（短）**：

> 本产品用于健康管理与饮食建议，不替代医疗诊断或治疗。如有不适或指标异常，请及时咨询医生/营养师。

### 4.2 新手建档（分步，避免卡住）

#### Step1 基础信息（必填最少）

- 身高、体重（必填）
- 年龄段、性别（建议必填）
- 运动水平（久坐/轻/中/高）
- 自动计算：基础代谢/建议能量区间（内部用）

#### Step2 选择改善目标（至少选 1）

- 减脂 / 高血压 / 高尿酸 / 脂肪肝 / 高血脂 / 抗炎（可多选）
- 选择一个“主目标”（用于周报主展示）

#### Step3 指标补充（条件必填，可跳过但强提醒）

- 高血压：血压（建议必填，可稍后补）
- 高尿酸：尿酸（可选但强建议）
- 脂肪肝：肝功能（可选）
- 高血脂：血脂（可选）
- 统一策略：允许“稍后补充”，但进入周报或目标建议页持续提醒

#### Step4 过敏/忌口（可选）

- 过敏原（多选）
- 忌口/不吃（自由输入或标签）

**验收**：

- 未完成身高体重无法进入首页。
- 主目标必须有且仅一个。
- “稍后补充”不阻断，但在对应目标页持续提示。

### 4.3 首页（今日概览）

**内容**：

- “拍照记录”主按钮（置顶）
- 今日摘要卡（按主目标选显示重点）
  - 通用：今日已记录餐次、估算热量
  - 目标相关：例如高血压显示钠风险提示；减脂显示能量区间提示
- 今日餐次列表入口
- 周报入口（本周进度：已记录 X 天）

**验收**：

- 无记录时显示引导：“拍一张包含多菜的照片，系统自动识别”。
- 有记录时展示今日汇总。

### 4.4 拍照/上传页

- 拍照/相册选择
- 提示文案（短）
  - “尽量包含整盘/整桌，光线充足”
  - “可拍一张多菜，一次记录一餐”

**验收**：

- 失败重试提示。
- 上传成功进入“识别中”状态页（带进度）。

### 4.5 识别结果页（核心页）

**页面结构（从上到下）**：

1. 本餐汇总（总热量 + 目标关键项：钠/糖/脂肪风险等）
2. 结论标签：✅适合 / ⚠️谨慎 / ❌不建议（温和措辞）
3. 3 条行动建议（短句、可执行）
4. 菜品列表（N 个），每项展示：
   - 菜名（可点编辑）
   - 做法推断（小字）
   - 份量：自动估算（可微调）按钮
   - 删除按钮
5. 底部固定：保存本餐

**状态与兜底**：

- 识别不确定：提示“我可能没认准，你可以点菜名修改”。
- 若只识别到 1 类：允许用户“新增菜品”。
- 若做法置信度低：在编辑页提供“做法选择”（可选）。

**验收**：

- 不做任何微调/编辑，也能直接保存成功。
- 支持“删除某道菜”“新增菜品”“修改菜名”。
- 点击微调后，汇总营养与建议实时刷新。

### 4.6 菜品编辑（弹窗/子页）

**功能**：

- 修改菜名（推荐列表 + 搜索）
- 修改做法（蒸/煮/炒/煎/炸/凉拌/烤/汤）
- 备注（可选）

**验收**：

- 修改后回到识别页并触发重算。

### 4.7 份量微调（底部抽屉）

**入口**：每道菜“份量：自动估算（可微调）”

- Tab A：大/中/小
  - 默认选中“中（自动）”
  - 选择后立刻预览变化（不关闭抽屉）
  - 倍率（默认）：小 0.75x / 中 1.0x / 大 1.30x

- Tab B：手掌参照
  - 选择类型（自动猜测 + 可改）：主食/蛋白/油脂/其他
  - 选择数量：0.5 / 1 / 2 / 3（掌/拳/拇指的对应单位）
  - “应用并更新”

**验收**：

- 应用后写入 final_grams，更新汇总与建议。
- 可恢复“自动估算”。

### 4.8 记录页（按天/餐次）

- 日历或上下翻天
- 餐次分组：早餐/午餐/晚餐/加餐
- 每餐卡：缩略图、总热量、结论标签、关键项（按目标）
- 操作：复制一餐（历史复用）、删除、查看详情

**验收**：

- 支持复制一餐到当天同餐次（默认时间为当前）。
- 删除有二次确认。

### 4.9 周报页（按主目标 3 指标）

**免费版（必须）**：

- 本周概览：记录天数、能量达标率
- 主目标 3 指标（不同目标不同指标）
- 3 条下周行动建议（短句）

**Plus 版（解锁）**：

- 深度洞察
  - Top 贡献食物/餐次（例如“钠最高的 3 类来源”）
  - 风险分布（晚餐/外卖占比）
  - 改善建议：替代选择、点餐建议、搭配建议

**验收**：

- 主目标不同，周报指标自动切换。
- 多目标用户：主目标完整展示，次目标展示简版卡片（1-2 条结论）。

### 4.10 我的（档案/指标/Plus）

- 档案编辑：目标、过敏忌口、运动水平
- 指标录入
  - 体重（必填，可设提醒但 MVP 不强做提醒）
  - 血压（当主目标/目标包含高血压时突出显示）
  - 其他指标按目标展示
- Plus 订阅：权益说明、订阅入口、恢复购买
- 协议/隐私/免责声明/反馈

**验收**：

- Plus 订阅状态可正确反映到周报与详细分析入口。

## 5. 周报指标规则（MVP 内置）

- 通用：能量达标率/热量趋势（所有人都有）

**主目标 3 指标**：

- 减脂：能量达标率 + 蛋白达标率 + 蔬菜/纤维达标率
- 高血压：能量达标率 + 钠（或高钠餐次数） + 蔬果频次评分
- 高尿酸：能量达标率 + 高嘌呤次数/评分 + 含糖饮料/酒精次数
- 脂肪肝：能量达标率 + 添加糖/甜饮次数 + 油炸/高脂次数
- 高血脂：能量达标率 + 饱和/反式脂肪风险 + 纤维/全谷频次
- 抗炎：能量达标率 + 超加工食品次数 + 抗炎食物摄入评分

## 6. 数据模型（MVP 最小字段）

- **User**：openid、unionid、created_at、plus_status、plus_expire_at
- **Profile**：user_id、height、weight、age_group、sex、activity_level、goals[]、primary_goal、allergies[]、avoid_foods[]
- **UserMetric**：user_id、type(weight/bp/ua/lipid/…)、value、measured_at
- **Meal**：meal_id、user_id、time、meal_type、image_url、summary_nutrition(json)、conclusion_tag、advice_list[]、confidence
- **Dish**：dish_id、meal_id、name、category、cook_method、base_grams、final_grams、adjust_method(auto/size/palm)、nutrition(json)、confidence
- **DishTemplate（库）**：template_id、name、default_base_grams、ingredients[]（ingredient_id + ratio）、cook_method_defaults、risk_rules
- **Ingredient（库）**：ingredient_id、name、nutrients_per_100g(json)
- **WeeklyReport**：user_id、week_start、week_end、primary_goal、metrics(json)、advice[]、insights(json, plus)

---

## 7. 接口（前后端契约，MVP）

- `POST /analyzeMeal`（上传图片）
  - 返回：`dishes[]`、`summary`、`conclusion`、`advice`、`confidence`
- `POST /recalcMeal`（传 dishes 修正/份量倍率）
  - 返回：更新后的 `summary` / `conclusion` / `advice`
- `POST /saveMeal`（落库）
- `GET /dayLog?date=`
- `GET /weeklyReport?week=`
- `POST /metric`（录入体重/血压等）
- `POST /purchase/verify`（微信支付回调/验签/发放权益）

---

## 8. 风控与合规（MVP 要做的最低限）

- 明确“健康管理工具”定位与免责声明（注册、我的页可见）。
- 不给诊断结论；不输出“停药/用药”建议。
- 对明显异常指标（用户录入）提示“请咨询医生”。
- 健康数据隐私：用途说明、最小化采集、可删除账号数据（MVP 可先做“申请删除入口 + 人工流程”，后续自动化）。

---

## 9. 埋点与数据看板（MVP 最关键）

- 建档完成率（分步骤流失点）
- 拍照→识别成功率、识别后保存率
- 纠错率、微调率（大中小/手掌）、保存耗时
- 周报打开率、Plus 解锁点击率、付费转化率
- 留存：D1/D7、周活跃、连续记录天数

## 10. 研发任务拆解（按泳道）

建议分两期：P0 闭环上线（2~4 周节奏）→ P1 增强识别与 Plus 洞察（再 2~4 周）

### A. 前端（小程序）

#### P0 必须

- 登录授权 + 协议确认
- 建档四步页（含“稍后补充”与目标绑定提示）
- 首页/记录/周报/我的 Tab 框架
- 拍照上传 + 识别中状态页
- 识别结果页（多菜列表、保存、编辑、删除、微调入口）
- 份量微调抽屉（大中小 + 手掌参照）
- 记录页（日历/餐次列表/复制/删除）
- 周报页（免费版）
- 我的页（档案编辑、指标录入、协议与免责声明）

#### P1 增强

- Plus 深度洞察展示
- 更细的错误引导与空状态运营文案
- 体验优化：历史菜品快捷添加、常用纠错词典

### B. 后端（API + 存储 + 支付）

#### P0 必须

- 用户与档案服务（Profile CRUD）
- 图片上传（对象存储）与签名 URL
- Meal/Dish 存储（含可重算）
- 周报生成（批处理/按请求生成均可）
- 指标录入（体重、血压等）
- 基础权限与数据隔离（按 user_id）

#### P1 增强

- Plus 订阅与权益系统（微信支付、验签、过期处理、恢复购买）
- 数据删除能力（用户申请删除）
- 周报洞察计算（Top 贡献、餐次分布等）

### C. 算法/AI（先可用，再变准）

#### P0 必须（可用优先）

- A 层：菜品/品类识别（多标签输出）
- B 层：模板拆解（菜品→食材→营养）
- C 层：做法推断（蒸/煮/炒/炸… 先粗分类）
- 份量自动估算：按模板默认 base_grams
- 重算服务：接收倍率/手掌换算→输出营养与建议
- 置信度策略：低置信度提示用户可纠错

#### P1 增强

- 多菜分割/更稳的多菜检测
- 油盐糖风险评分更精细（结合轻问答或 Plus 增强）
- 纠错数据闭环：将用户修正用于模板/识别优化

### D. 数据（营养库/模板库/规则）

#### P0 必须

- 通用食材库（先覆盖高频食材）
- 常见菜品模板库（建议 100–200 道高频家常 + 外卖）
- 目标规则：
  - 各目标的“风险成分优先级”（例如高血压优先钠）
  - 结论标签规则（适合/谨慎/不建议）
  - 3 条行动建议模板（按目标 + 做法 + 风险项生成）

#### P1 增强

- 模板精细化（地区口味差异、外卖常见菜）
- “超加工/抗炎食物”标签体系更完善

## 11. 里程碑建议（按交付物）

### 里程碑 1：闭环可用

建档完成 → 拍照识别 → 自动估算 → 保存 → 记录页可回看

### 里程碑 2：微调与周报

份量微调即时重算；周报按主目标展示 3 指标 + 建议

### 里程碑 3：Plus

详细分析与深度洞察展示；微信支付打通 + 权益校验
